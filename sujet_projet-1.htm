    Written Project: The project purpose (maximum 3 students in group) is to reproduce the game of Power 4 on a set of web pages. The website will have a presentation page, a rules page and a game page. The game page may use other pages or pop-up ads. The architecture and design of the web pages are at your convenience, but a minimum of 3 navigable pages between them is required. The game power 4 is played by 2 players with counters and a game grid. The counters have a unique color for each player, for example:

    • Yellow for player 1
    • Red for player 2

    On a vertical playing grid, players in turn place their checkers. As the grid is considered vertical, gravity is imposed in the dynamics of the game. That is, if the player places his pawn on top of a column, the pawn falls to the bottom of the column or to the top of the last pawn in the column. The game is played with the mouse and the keyboard.

    It will be possible to name/click directly on a valid square to place your pawn. If a column is full, it will not be possible to place a pawn. The principle of the game is to make lines of 4 tokens.

    The game ends:

    1. With a winner, when there is a row, a column or a diagonal of 4 pieces of the same color. At that moment, the player who holds the color wins.
    2. By a draw between the 2 players, if all the slots of the game grid are filled with no possible row.

    The architecture and design of the game pages are at your convenience, however you must respect the display areas:

    1. A zone for the menu that allows you to start the game. This zone allows you to choose the number of boxes in the game grid (Rows * Columns).
    2. A zone for the game grid.
    3. A zone for the scores.
    4. A zone for determining the colors of the pieces.
    5. A zone for a stopwatch.

    For example:

    Here's how a game unfolds for a player:

    1. Choose a game grid (It must be valid)
    2. Choose a display mode (the color of the counters for example)
    3. End of the game:
      1. When a line is found = Award points to the winner + update of the winner's score + success message + time (A bonus point can be awarded if the time is low)
      2. When the grid is filled without any possible line = Award points to the 2 players + update of the score of the 2 players + draw math message + time
